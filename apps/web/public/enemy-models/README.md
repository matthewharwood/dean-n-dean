# Enemy 3D models

Low-poly textured GLB models that the gathering monster card renders in place of the
2D `enemies/<id>.webp` portrait. Generated by the Tripo3D P-Series pipeline:

```bash
bun run tripo:enemy-model <enemy-id>      # see /tripo-enemy-model command
```

Each registered model (`gathering-enemy-models.ts`) is shown via `GlbMonsterViewer`
(Three.js, WebGPU-preferred → WebGL2 fallback, moody bioluminescent lighting).

## Notes

- `hadal-glow-polyp-echo_L1.glb` is the real Tripo-generated model (the L1 poster
  variant). `hadal-glow-polyp-echo.glb` (the base / loop-0 form) is currently a copy
  of it, so both forms show the real model. To give the base its own distinct sculpt,
  run `bun run tripo:enemy-model hadal-glow-polyp-echo` and overwrite it — no code
  changes needed (the registry already lists both basenames).
- Rendering uses `setAnimationLoop` + `renderAsync` on WebGPU (a synchronous
  `render()` before WebGPU's pipelines compile paints a black frame); WebGL renders
  synchronously. See `apps/web/app/canvas/use-glb-monster-viewer.ts`.
